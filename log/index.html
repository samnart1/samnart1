TCP Echo Server

C++

Building a multi-connection TCP server from scratch using Linux socket APIs. No frameworks, no libraries—just raw sockets and syscalls.

This week's focus

Getting epoll working with non-blocking sockets. The goal is to handle thousands of concurrent connections without threading overhead.

Decisions & learnings

  • Chose epoll over select/poll—better scalability for high connection counts
  • Going with edge-triggered mode to reduce syscall overhead
  • Beej's Guide is still the gold standard for this stuff

Stuck on

Handling partial reads/writes correctly with edge-triggered epoll. Need to drain the buffer completely or risk missing events.
